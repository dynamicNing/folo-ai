# ADR-005: public/ 构建产物提交进 Git，服务器免构建部署

- **日期**：2026-04-16
- **状态**：已接受

## 背景

项目同时作为开发目录和生产部署目录使用。需要决定服务器是否需要 Node 构建环境。

## 决策

`public/` 目录（Vite 构建产物）提交进 Git。服务器部署只需 `git pull && npm install --production`，不需要执行 `npm run build`。

## 原因

- 服务器无需安装 devDependencies（vite、vue 等），减少服务器环境依赖
- 部署流程更简单，`git pull` 即生效
- 开发者本地 build 后提交，产物经过本地验证再上线

## 后果

**正面**：服务器零构建依赖；部署命令极简。  
**负面**：Git 仓库包含编译产物，体积略增；每次前端改动需本地 build 后再提���。  
**中性**：需要团队约定：前端改动必须先 `npm run build` 再提交，否则服务器代码与源码不一致。

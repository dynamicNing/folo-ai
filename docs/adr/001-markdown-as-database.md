# ADR-001: 使用 Markdown 文件作为数据存储

- **日期**：2026-04-16
- **状态**：已接受

## 背景

项目初期需要一个内容存储方案。内容以文章为主，写入频率低，读取为主，部署在单台服务器上，维护者只有一人。

## 决策

使用 Markdown 文件（带 frontmatter）作为唯一数据源，不引入数据库。文件按分类存放在 `content/<category>/` 目录下，文件名作为 slug。

## 原因

- 内容天然适合 Markdown，写作体验好
- 无需运维数据库，部署简单
- 文件可直接用 Git 管理，天然版本历史
- 内容量小（个人站点），文件扫描性能够用

## 后果

**正面**：零数据库依赖，部署极简；内容可离线编辑；Git history 即变更记录。  
**负面**：不适合高频写入；无法做复杂查询（全文搜索需另行实现）；并发写入存在风险。  
**中性**：如未来内容量或并发需求增长，需迁移到 SQLite 或 PostgreSQL，迁移成本低（frontmatter 字段可直接映射为表字段）。

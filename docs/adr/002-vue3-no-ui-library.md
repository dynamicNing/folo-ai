# ADR-002: 前端使用 Vue3 + Vite，不引入 UI 组件库

- **日期**：2026-04-16
- **状态**：已接受

## 背景

需要选择前端技术栈。项目是个人站点，页面数量少，设计风格强调轻量和自定义。

## 决策

使用 Vue3 + Vite 作���前端框架，状态管理用 Pinia，路由用 Vue Router。不引入 Element Plus、Ant Design Vue 等组件库，全部用纯 CSS 实现。

## 原因

- Vue3 Composition API 代码组织清晰，适合长期维护
- Vite 构建速度快，开发体验好
- 不引入组件库：包体积更小；样式完全可控，便于主题定制；避免组件库版本升级带来的维护负担

## 后果

**正面**：产物体积小；CSS 变量实现主题切换代价极低；无第三方组件库升级依赖。  
**负面**：表格、弹窗、表单等需要自己实现，初期开发量略多。  
**中性**：如后台功能复杂度上升（如富文本编辑器、复杂表单），可局部引入组件库，不影响现有代码。

# Skill Workbench 简版说明

> 面向产品、业务、运营或项目协作方

**版本**：v0.2  
**日期**：2026-04-25  
**适用对象**：想快速了解“这套东西现在能做什么”的非开发同学

---

## 1. 这是什么

Skill Workbench 是 `folo-ai` 后台里的一个内部模块，用来统一管理和调用一部分 AI skill。

它现在主要解决两件事：

- 把系统内置的 skill 做成可在 Web 后台直接调用的能力
- 把本机已经安装的外部 skill 纳入统一目录，方便查看、整理和后续接入

它不是技能市场，也不是开放给普通用户的产品功能，而是一个内部管理台。

---

## 2. 现在能做什么

### 已经可以做

- 在后台查看所有已纳管的 skill
- 区分“内置 skill”和“外部 skill”
- 对内置 skill 发起运行
- 查看运行过程、审批、结果、产物和失败原因
- 在后台配置 Anthropic 路由
- 在后台直接测试 Anthropic 代理是否可用

### 现在还不能做

- 不能让外部 skill 直接运行
- 不能在浏览器里直接拿模型密钥去请求第三方
- 不能把它当成通用脚本执行平台
- 不能从远程仓库一键安装 skill

一句话概括：**现在已经能“管理所有 skill”，但只有“内置 skill”能直接跑。**

---

## 3. 页面上会看到什么

### `/admin/skills`

这是 skill 总目录。

可以看到：

- 名称
- 来源
- 分类
- 版本或默认模型
- 更新时间

也可以按“内置/外部”筛选，并手动同步本机 skill 目录。

### `/admin/skills/:slug`

这是 skill 详情页。

会看到：

- 这个 skill 从哪里来
- 默认模型是什么
- 是否允许网络、文件、shell、浏览器等能力
- 输入输出结构

如果是内置 skill，会有运行入口；如果是外部 skill，会明确提示“当前只做目录管理”。

### `/admin/runs`

这是运行记录页，用来追踪：

- 谁跑了哪个 skill
- 用了哪个模型
- 成功还是失败
- 跑了多久

### `/admin/settings/models`

这是 Anthropic 路由配置页。

适合配置：

- 官方 Anthropic
- 兼容 Anthropic 的代理站

### `/admin/settings/anthropic-debug`

这是 Anthropic 调试页，用来直接验证：

- 当前代理地址能不能通
- 鉴权是不是对的
- 返回格式是否兼容系统

---

## 4. 这套东西为什么有价值

### 对业务

- 把零散的 skill 集中起来，避免“只有命令行能用”
- 把运行结果留痕，便于复盘
- 对高风险操作加入审批，降低误用风险

### 对团队

- 内置 skill 和外部 skill 都能进入同一个目录
- 后续想给某个外部 skill 接执行能力时，不用从零开始找来源
- Anthropic 代理是否可用，可以在后台直接验证，不必反复查命令行配置

---

## 5. 当前产品定位

这不是一个“开放平台”，而是一个“内部 AI 工具管理台”。

更准确地说，它现在有两个层次：

- 第一层：可运行的内置 skill
- 第二层：可管理的外部 skill 目录

这个定位是刻意控制过的，目的是先把“管理、审计、配置、调试”做好，再决定哪些外部 skill 值得真正接进运行链路。

---

## 6. 当前边界

下面这些边界是当前版本故意保留的，不是遗漏：

- 外部 skill 先纳管，不直接运行
- 只有管理员可以使用
- 模型密钥只保存在服务端
- Anthropic 路由优先解决“后台可配 + 可调试”

这样做的好处是节奏更稳，也更适合当前项目阶段。

当前版本还有两个明确约束：

- 外部 skill 需要管理员手动点一次“同步目录”才会出现在后台
- Anthropic 设置里的“数据库覆盖”关闭后，实际调用会回退到环境变量或内置默认

---

## 7. 下一步最值得做什么

如果继续往前推进，优先级建议是：

1. 给外部 skill 增加“运行方式绑定”
2. 给外部 skill 增加分组、标签、启停状态
3. 去掉开发期 hardcoded 配置，完全收敛到环境变量或后台配置
4. 增加 catalog 同步日志和版本追踪

---

## 8. 一句话结论

Skill Workbench 现在已经是一个可用的后台模块：

- 内置 skill 可以运行
- 外部 skill 可以统一管理
- Anthropic 代理可以在后台配置并调试

它的重点不是“把所有 skill 都跑起来”，而是先把整套管理和执行基础设施搭稳。
